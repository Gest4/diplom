Модуль 2. Конфигурация сервисов и служб на базе ОС Microsoft Windows Server 2019

15. Настройте параметры BIOS компьютера для работы с ПО виртуализации;

Для этого нужно выключить компьютер, и при включении нажать комбинацию клавиш для запуска BIOS, наиболее распространенная клавиша "DEL" или "F2". После этого открыть вкладку «Advanced», дальше найти строку «Virtualization» или подобную и нажать «Enter». После проделывания нажать "F10" и "Enter".

16. Установите ПО Vmware Workstation;

Для этого на компьютере должен быть установщик "Vmware Workstation", нужно установить в удобное место на диске.

17. Создайте ВМ со след. параметрами: имя ВМ - HEAD-DC; виртуальный диск в виде одного файла объемом 40 гб; объем ОП 4 гб; 4 ядра процессора; виртуальный Intel VT-x/EPT или AMD-V/RVI; тип подключения сетевого адаптера - VMnet0 (автомост);

Нужно запустить VMware workstation, после в главном меню выбрать "Новая виртуальная машина", после "Выборочный (дополнительно)", потом появится окно с "Совместимым оборудованием", далее появится окно с выбором режима установки OC, выбираем "Я установлю операционную систему позже" и далее, потом выбираем "Microsoft Windows", в следующем окне выбираем папку установки нажимаем далее, в окне "Тип прошивки" ставим "BIOS" и переходим далее, откроется следующее окно "Конфигурация процессора" ставим кол-во ядер 4 нажимаем далее, в следующем окне от нас требуется выставить количество оперативной памяти ставим 4 гб жмём далее, в следующем окне "Тип сети" выбираем "Не использовать сетевое соединение", в окне "Выбор типов контролёров ввода/вывода" оставляем всё как есть и жмём "Далее", в следующем окне "Выбор типа диска" опять оставляем всё как есть жмём далее, в окне "Выбор диска" выбираем "Создать новый виртуальный диск" (стоит по умолчанию) и переходим далее, в окне "Указание размера диска" указываем 40 гб и "Сохранить виртуальный диск в одном файле" жмём далее, потом подтверждаем жмём далее, нажимаем "Настройка оборудования" заходим в сетевой адаптер выбираем "VMnet0 (автомост)" затем закрыть и готово. Если на каком-то из этапов установки произошла ошибка, то это можно исправить, нажав "Изменить настройки" выбрав виртуальную машину.

18. Установите ОС Windows Server 2019; пароль учетной записи Administrator: Ab!@dmin;

Выбрав виртуальную машину нажимаем "Изменить настройки", выбираем "CD/DVD драйвер", затем «Использовать файл образа ISO», выбираем iso образ диска с ОС, нажимаем "ОК". Запускаем виртуальную машину, если всё сделано правильно запускается установка Windows Server. Устанавливаем с GUI, если написано в задании CORE, ставим CORE. После установки Windows Server ставим пароль "Ab!@dmin".

19. Задайте имя серверу - HEAD-DC; настройте сетевые параметры согласно Схеме;

Нужно ввести комбинацию "win"+"r", ввести "cmd" нажать "ок", в командной строке ввести (wmic computersystem where name="%computername%" call rename name="HEAD-DC") или через свойства компьютера, изменить параметры, изменить.

20. Сделайте сервер контроллером домена - AbiMoscow.com. Создайте организационные единицы, группы и пользователей в домене AbiMoscow.com согласно таблице;

Открываем Sever Manager. Manage > Add Roles and Features, В разделе Server Roles находим роль Active Directory Domain Services, отмечаем. Подтверждаем установку компонентов кнопкой "Install". Жмём на ссылку "Promote this server to domain controller", в разделе "Result". Выбираем "Add a new forest" и вводим домен "AbiMoscow.com". В следущем окне устанавливаем флажок напротив "DNS", и вводим пароль Ab!@dmin, затем прожимаем "next" до окна "Install". Запускаем Active Directory Users and Computers (dsa.msc). Users, New Object > User, заполняем согласно таблице.

21. Установите и настройте службы DNS и DHCP для сети LAN1. Служба DHCP должна иметь пул из 7 (семи) адресов, начиная с 16-го доступного адреса из сети LAN1, исключая 4 и 5 адрес; 3-ий адрес диапазона DHCP должен быть зарезервирован для машины HEAD-RDS. В опциях должно передаваться только шлюз и DNS сервер;

22. Настройте зону прямого и обратного просмотра DNS, а также добавьте необходимые записи (A) для серверов. Сконфигурируйте пересылку на адреса: 1.1.1.1 и 8.8.8.8;

23. Создайте ВМ со след. параметрами: имя ВМ – HEAD-RDS; виртуальный диск в виде одного файла объемом 40 гб; объем ОП 4 гб; 4 ядра процессора; виртуальный Intel VT-x/EPT или AMD-V/RVI; тип подключения сетевого адаптера – сетевой мост;

24. Установите ОС Windows Server 2019 пароль учетной записи Administrator: Ab!@dmin ;

25. Задайте имя серверу - HEAD-RDS; ОС должна получать зарезервированный IP-адрес от MAIN-DC согласно Схеме 1;

26. Введите сервер в домен AbiMoscow.ru. В структуре домена переместите его в подразделение HEAD;

Нужно зайти в командную строку, ввести "sconfig", затем выбрать 1 пункт, затем ввести "d" и ввести имя домена(AbiMoscow.com),

27. На сервере HEAD-RDS сконфигурируйте: - терминальный сервер с лицензированием по компьютерам (используйте временную лицензию); - веб доступ RemoteApp к службам терминалов сервера; - публикацию программы «WordPad» на веб-портале RemoteApp для всех сотрудников отдела BO_IT;

28. Обеспечьте отказоустойчивое хранилище на сервере HEAD-DC создав зеркальный рейд, добавив два диска по 2GB. Назначьте ему букву (D:);

29. На сервере HEAD-DC создайте сетевые папки, настройте фильтры блокировки файлов (запретить хранение файлов аудио и видео) в соответствии с таблицей;

30. Настройте и примените групповые политики к пользователям и клиентским рабочим станциям домена;

31. Создайте политику учетных записей для всех пользователей домена AbiMoscow.com: - вести журнал паролей - 7; - максимальный срок действия пароля - 60; - пароль должен отвечать требованиям сложности - выключено; - минимальная длина пароля – 8; - продолжительность блокировки учетной записи - 5; - пороговое значение блокировки - 3; - время до сброса счетчика блокировки - 2;

32. Добавьте пользователей отдела BRANCH1/BO_IT в локальную группу администраторов для всех компьютеров (ноутбуков) домена AbiMoscow.com;

33. Подключите сетевые диски сотрудникам отделов Sales и IT, назначьте букву (G:);

34. Включите удаленный рабочий стол на всех компьютерах, находящихся в филиале (Branch1);
